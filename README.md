# 🏥 Patient Calendar App

A clean, efficient web application built for doctors to manage and schedule patient appointments — designed as a better alternative to Google Calendar for medical use.

> Built with love for a real doctor. 💙

---

## ✨ Why This Project Exists

Google Calendar works for general scheduling, but it falls short for medical practices:

- No patient context on calendar blocks
- Messy, unorganized weekly view when many appointments exist
- No condition or notes attached to time slots
- No way to track appointment status (done, cancelled, etc.)
- No patient list or search

This app solves all of that in one clean interface.

---

## 🖥️ Features

- **Weekly Calendar View** — clean time-grid layout, click any slot to add an appointment
- **Patient Management** — add patients with name, phone, condition, and notes
- **Smart Appointment Blocks** — color-coded by doctor's choice, shows patient name + condition
- **Status Tracking** — mark appointments as Scheduled, Done, or Cancelled
- **Edit & Delete** — click any block to edit or remove it
- **Patient Search** — instantly filter patients by name or condition
- **Stats Dashboard** — see today's count, this week's total, completed, and cancelled
- **Upcoming Panel** — quick view of next scheduled appointments
- **Mobile Responsive** — works on phone and tablet via browser
- **Motivational Quotes** — a small touch of encouragement on every session 🌟

## 🚀 Getting Started (Local Development)

### Prerequisites
- Python 3.10+
- Node.js 18+
- Git

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/patient-calendar.git
cd patient-calendar
```

### 2. Start the Backend
```bash
cd backend

# Create and activate virtual environment (Windows)
python -m venv venv
venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Run the server
uvicorn main:app --reload
```

Backend runs at: `http://localhost:8000`
API docs available at: `http://localhost:8000/docs`

### 3. Start the Frontend
```bash
cd frontend
npm install
npm start
```

Frontend runs at: `http://localhost:3000`

## 🌐 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/patients` | Get all patients |
| POST | `/patients` | Create a new patient |
| DELETE | `/patients/{id}` | Delete a patient |
| GET | `/appointments` | Get all appointments |
| POST | `/appointments` | Create an appointment |
| DELETE | `/appointments/{id}` | Delete an appointment |

Full interactive API documentation: `/docs` (Swagger UI, auto-generated by FastAPI)

## 📦 Deployment

- **Frontend** → [Vercel](https://vercel.com) — import repo, set root to `frontend/`
- **Backend** → [Railway](https://railway.app) — connect repo, point to `backend/` folder

After deploying backend, update the `API` constant in `frontend/src/App.tsx`:
```tsx
const API = "https://your-railway-url.up.railway.app";
```
